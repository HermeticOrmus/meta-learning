# Meta Learning

> **A practical system for learning how to learn, distilled from history's greatest polymaths and modern cognitive science.**

---

## Why This Exists

Most people learn inefficiently. They:
- Watch tutorials endlessly without building (tutorial hell)
- Memorize facts without understanding connections
- Study in isolation without feedback
- Sprint heroically then burn out

**Exceptional learners do the opposite.** They discover and apply specific practices that compound over time. These practices are learnable.

This repo extracts those patterns from:
- **Historical Polymaths**: Leonardo da Vinci, Nikola Tesla, Benjamin Franklin, Richard Feynman, Aristotle, John von Neumann
- **Modern Learning Science**: Barbara Oakley, Anders Ericsson, Scott Young, Dr. Justin Sung, Carol Dweck

The result: A practical system you can apply immediately.

---

## Core Philosophy

### Build, Don't Collect

```
Learn TO ➜ Build things that solve real problems
Not TO ➜ Collect certificates, badges, or superficial familiarity
```

### The Learning Equation

```
Understanding + Application + Teaching = Mastery
      ↑              ↑            ↑
 Mental Models    Projects    Explain to Others
```

### Sustainable Rhythm Over Heroic Sprints

30 minutes daily beats 8-hour weekend marathons. Consistency compounds.

## Quick Start (10 minutes)

### 1. Choose ONE Skill
Ask yourself:
- Does this solve a real problem I have?
- Can I commit 30 min/day for 4-8 weeks?
- Will I build something real with this?

### 2. Copy the Template
```bash
cp templates/learning-sprint.md my-skill/README.md
```

### 3. Fill Out Your "Why"
Be honest. "Looks cool" isn't enough. What real problem does this solve?

### 4. Build Something Today
Not tomorrow. Today. Even "Hello World" counts. Ship something.

### 5. Log It
```bash
cp templates/progress-journal.md my-skill/progress.md
# Log what you built, what you learned, what blocked you
```

## The Master Learners

### Leonardo da Vinci
**Method**: Observe deeply → Question everything → Connect across domains → Build to understand

*"Saper vedere"* (knowing how to see) - Leonardo filled notebooks with observations, always drawing to understand.

### Nikola Tesla
**Method**: Visualize deeply → Process in solitude → Think from first principles → Focus obsessively

Tesla ran complete "mental experiments" before building anything physical.

### Benjamin Franklin
**Method**: Track progress daily → Teach what you learn → Seek harsh feedback → Apply immediately

Franklin's "virtues chart" was deliberate practice decades before the term existed.

### Richard Feynman
**Method**: Explain simply → Find gaps → Learn gaps → Play with concepts

The Feynman Technique: If you can't explain it simply, you don't understand it.

## Evidence-Based Methods

### Encoding Over Reviewing (Dr. Justin Sung)
Spend 70% of time **understanding**, 30% reviewing. Most people invert this.

### Focused + Diffuse Modes (Barbara Oakley)
Alternate intense focus (25 min) with relaxed breaks. Both modes are necessary.

### Deliberate Practice (Anders Ericsson)
1 hour of deliberate practice > 10 hours of comfortable repetition.

### Directness Principle (Scott Young)
Learn by doing the thing, not by preparing to do the thing.

### Spaced Repetition (Ebbinghaus)
Review at increasing intervals: 1 day → 3 days → 7 days → 14 days → 30 days.

## Common Patterns Across All Masters

1. **Obsessive Curiosity** - Ask "why?" constantly
2. **Cross-Domain Thinking** - Connect ideas from different fields
3. **Learn by Making** - Hands-on > passive study
4. **Teach to Learn** - Explaining clarifies understanding
5. **Systematic Tracking** - Journals, notebooks, documentation
6. **First Principles** - Build from fundamentals, question assumptions
7. **Deep Work + Rest** - Intense focus + integration periods

## Daily Workflow

### Practice Session (30 min)

**Review** (2 min)
- What did I learn yesterday?
- What's today's specific goal?

**Build** (25 min)
- Focus on doing, not consuming
- Push slightly beyond comfort zone

**Log** (3 min)
- What did I build?
- What did I learn?
- What blocked me?
- What's tomorrow's goal?

### Weekly Review (30 min)

- Days practiced this week
- Projects shipped
- Knowledge extracted
- What worked? What didn't?
- Next week's focus

## Anti-Patterns to Avoid

### Tutorial Hell
**Symptom**: Watching endlessly, building nothing
**Fix**: 20% consume, 80% build. Ship today.

### Shiny Object Syndrome
**Symptom**: Starting new skills before mastering current ones
**Fix**: Max 2 active skills. Complete before adding.

### No Feedback Loop
**Symptom**: Practicing alone, reinforcing bad habits
**Fix**: Weekly code reviews, peer feedback, public building.

### Perfectionism Paralysis
**Symptom**: Never shipping because "not ready"
**Fix**: Ship imperfect. Iterate. "Done" beats "perfect".

### Burnout Cycles
**Symptom**: Hero sprints → exhaustion → quit
**Fix**: 30 min daily. Rest days. Sustainable rhythm.
