# Compositional Learning Patterns

> How learning methods combine, transform, and transfer across domains

---

## The Core Insight

Learning methods aren't isolated techniques. They **compose**. Understanding how they fit together lets you build custom learning strategies for any situation.

```
Method A + Method B = New capability neither has alone
```

This document explores the algebra of learning.

---

## Method Composition

### Sequential Composition (A → B)

Do A, then use its output as input to B.

**Feynman → Franklin**
```
Explain simply (find gaps) → Track gaps daily → Systematic elimination
```
Feynman finds WHAT you don't know. Franklin's tracking ensures you ADDRESS it.

**Tesla → Leonardo**
```
Visualize solution → Observe deeply → Draw to verify → Build
```
Mental simulation first, then grounded observation to validate.

**Oakley → Ericsson**
```
Focused mode (25 min) → Diffuse (rest) → Deliberate practice (target weakness)
```
The Pomodoro provides structure. Deliberate practice provides direction.

### Parallel Composition (A ‖ B)

Run both simultaneously, different aspects of same learning.

**Franklin ‖ von Neumann**
```
Track daily progress    ‖    Consume widely, find patterns
(consistency)               (breadth)
```
Track your learning while expanding your knowledge base.

**Feynman ‖ Aristotle**
```
Explain simply    ‖    Classify systematically
(depth)               (organization)
```
Understand deeply while organizing what you learn.

### Feedback Composition (A ↔ B)

Each method informs the other in a loop.

**Build ↔ Learn**
```
Build something → Hit wall → Learn specific thing → Apply → Build more
```
Just-in-time learning with immediate application.

**Teach ↔ Understand**
```
Try to teach → Find gaps → Deepen understanding → Teach better → Find new gaps
```
Teaching and learning reinforce each other.

## The Method Algebra

### Identity Method
The "do nothing" method. Some learning happens passively (sleep consolidation, subconscious processing). This is the diffuse mode.

### Inverse Methods
Some methods undo the effects of others:

| Method | Inverse | Why |
|--------|---------|-----|
| Cramming | Spaced repetition | Massed practice creates illusion; spacing creates durability |
| Passive consumption | Active building | Watching erases; doing creates |
| Isolation | Teaching | Solo reinforces gaps; teaching reveals them |

### Commutative Pairs
Order doesn't matter:

```
Franklin + Feynman = Feynman + Franklin
(Track + Explain) = (Explain + Track)
```

Both sequences lead to: systematic gap identification and elimination.

### Non-Commutative Pairs
Order matters:

```
Tesla + Musk ≠ Musk + Tesla
(Visualize → Ship fast) ≠ (Ship fast → Visualize)
```

Visualize-then-ship creates considered action.
Ship-then-visualize creates reactive iteration.

Both are valid. Different outcomes.

## Learning Transformations

### Domain Transfer

Learning methods that work in one domain often transfer to another, but require **adaptation**.

**Deliberate Practice: Music → Programming**
```
Music: Isolate difficult passage, slow down, repeat with variation
Code:  Isolate difficult concept, simplify context, practice edge cases
```

The STRUCTURE transfers. The CONTENT changes.

**Feynman Technique: Physics → Social Skills**
```
Physics: Explain quantum mechanics to a child
Social:  Explain your job to someone outside your field
```

Same method. Tests for jargon-hiding in any domain.

### Abstraction Levels

Methods operate at different levels:

```
Meta-level:    How I approach learning itself (this document)
Strategy:      Which methods to combine for this skill
Tactics:       How to execute a specific method today
Execution:     The actual practice session
```

Higher-level methods inform lower-level choices:
```
Meta (growth mindset) → Strategy (deliberate practice) → Tactics (25 min focus) → Execution
```

## Composition Patterns

### The Foundation Pattern
```
Mental Model → Practice → Feedback → Refine
     ↑___________________________|
```

Every effective learning loop contains:
1. **Conceptual understanding** (encoding)
2. **Active application** (practice)
3. **External input** (feedback)
4. **Integration** (refinement)

Missing any element degrades learning.

### The Depth Pattern
```
Surface → Depth → Integration → Transfer

Survey material → Deep study → Connect to known → Apply elsewhere
```

Priming → Encoding → Elaboration → Application

### The Rhythm Pattern
```
Sprint → Rest → Sprint → Rest → Review

Focus → Diffuse → Focus → Diffuse → Consolidate
```

Sustainable learning requires oscillation, not constant pressure.

### The Teaching Pattern
```
Learn → Attempt to teach → Find gaps → Learn gaps → Teach again
```

Teaching is the highest form of learning because it forces completeness.

## Method Selection Heuristics

Given a learning challenge, how do you choose which methods to compose?

### By Learning Phase

| Phase | Primary Method | Supporting Methods |
|-------|---------------|-------------------|
| New skill | Metalearning (Young) | von Neumann (consume), Aristotle (classify) |
| Building foundation | Mental models (Sung) | Leonardo (observe), Tesla (visualize) |
| Active practice | Deliberate practice (Ericsson) | Franklin (track), Oakley (rhythm) |
| Breaking through plateau | Feynman (simplify) | Tesla (first principles) |
| Mastery | Teaching | All methods integrated |

### By Problem Type

| Problem | Method Composition |
|---------|-------------------|
| "I don't understand" | Feynman → First principles → Build |
| "I can't remember" | Spaced repetition + Active recall |
| "I know it but can't apply" | Directness + Deliberate practice |
| "I'm stuck" | Diffuse mode + Cross-domain thinking |
| "I'm burning out" | Rhythm + Rest + Reduce scope |
| "I'm bored" | Play (Feynman) + New challenge |

### By Learning Style

Map your preferences to method combinations:

**Visual learner**: Leonardo (observe, draw) + Tesla (visualize)
**Verbal learner**: Feynman (explain) + Aristotle (document)
**Kinesthetic learner**: Build-first + Direct practice
**Social learner**: Teaching + Community + Feedback loops

## Composition Anti-Patterns

### Over-Composition
Using too many methods at once. Cognitive overload.

**Fix**: Pick 2-3 methods max per learning session.

### Incompatible Composition
Some methods fight each other:

```
Cramming + Spaced repetition  (contradictory)
Perfectionism + Ship fast     (contradictory)
Tutorial hell + Direct practice (contradictory)
```

**Fix**: Recognize the contradiction. Choose one approach.

### Missing Feedback Loop
All theory, no reality check.

```
Read → Read → Read → Read → ???
```

**Fix**: Every composition needs a feedback mechanism.

### Skipping Levels
Jumping to advanced methods without foundation.

```
Deliberate practice on concepts you don't understand
```

**Fix**: Mental model BEFORE deliberate practice.

## Building Your Personal Composition

### Step 1: Audit Your Current Approach
What methods do you naturally use? What's missing?

### Step 2: Identify Your Weakness
- Can't retain? → Add spaced repetition
- Can't apply? → Add directness
- Can't explain? → Add teaching
- Burning out? → Add rhythm/rest

### Step 3: Design a Composition
Combine 2-3 methods that address your weakness while building on your strength.

### Step 4: Test and Iterate
Run the composition for 1-2 weeks. Track effectiveness. Adjust.

## The Meta-Pattern

All master learners discovered the same meta-pattern:

```
Understand → Practice → Get Feedback → Integrate → Transfer → Teach
```

Their individual methods are different **implementations** of this pattern.

Leonardo implemented with observation and drawing.
Franklin implemented with virtue tracking and clubs.
Feynman implemented with simple explanations and play.

The pattern is universal. The implementation is personal.

**Your task**: Find your implementation of the universal pattern.
